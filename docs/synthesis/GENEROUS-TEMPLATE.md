# <Skill Or Family> Generous Synthesis

Use this when research is strong enough to make design judgments, but the
runtime wording is not ready to compress yet.

Generous synthesis is scaffolding. It may explain, compare, and preserve
reasoning. It should still be directional: by the end, the next compact
synthesis pass should be obvious.

## Purpose

What behavior this skill or family should make predictable.

## Source Pressure

The sources, books, papers, manuals, priors, or research packets that should
shape the skill.

For each source cluster, name only:

- leading words;
- rules;
- failure modes;
- evidence gates;
- stop/ask conditions.

## Candidate Behavior

What the agent should do differently after the skill changes.

## Leading Words

Compact terms that recruit upper-bound priors.

## Agent Execution Surface

How the agent will actually execute the behavior: read, inspect, test, edit,
review, stop, ask, delegate, or validate.

## Evidence Gates

What must be true before the agent can claim done.

## Failure Modes

Average-agent behavior this skill should prevent.

## Design Questions

Open choices that must be resolved before compact synthesis.

## Rejected Or Deferred Options

What should not move forward, and why.

## Verbose Draft Notes

Temporary explanatory prose. Use this to expose the behavior before the
controlled-language pass removes excess.

## Compression Handoff

End with the smallest useful handoff into compact synthesis:

- chosen behavior;
- strongest leading words;
- candidate runtime wording;
- blunt gates;
- what stays research-only;
- validation tasks.
