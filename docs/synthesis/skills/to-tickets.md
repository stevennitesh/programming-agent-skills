## Intended Contract

To Tickets transforms one bounded settled-source packet into an exhaustive,
dependency-ordered set of Ready-for-agent implementation tickets. It owns
slicing, coverage, blocker order, ticket readiness, execution profiles,
publication scope, safe failure Returns, and exactly one unstarted downstream
recommendation.

The source owner retains outcome, commitments, acceptance, scope, exclusions,
and domain meaning. Tracker and label contracts own transport, relationships,
roles, lifecycle mapping, and Mutation read-back. The engineering contract
owns shared Source Trace, proof, and state-boundary semantics.
`$repo-bootstrap` owns missing or incompatible setup. `$implement` owns one
selected ticket. `$parallel-implement` owns only an explicitly requested
top-level delivery run over a verified non-empty parent graph.

The active runtime does not invent product decisions, create a parent spec,
triage raw intake, implement or dispatch work, close tickets, install skills,
or perform Git delivery. It stops unchanged on setup or commitment-changing
source gaps, reports exhaustive no-ticket coverage when appropriate, and never
claims a complete graph after an unknown or mismatched mutation.

## Relationships

Relationship topology and trigger meaning remain unchanged:

| Relationship | Observable trigger and Return |
| --- | --- |
| `$to-spec` -> `$to-tickets` | A verified parent is ready for slicing; preserve parent intent and Source Trace |
| Improve/Audit packet -> `$to-tickets` | A settled multi-slice solution exists; preserve its complete accepted packet |
| `$implement` -> `$to-tickets` | Selected work is shaping-unready; return all readiness defects without repairing inside Implement |
| `$parallel-implement` -> `$to-tickets` | Graph, profile, or matrix state is incomplete; return one exhaustive repair packet |
| `$to-tickets` -> `$repo-bootstrap` | Required setup is missing or incompatible; name the surface and stop unchanged |
| `$to-tickets` -> `$implement` | One item, a serial constraint, uncertainty, uneconomic parallelism, or no explicit parent-delivery authority; recommend the first ready item and stop |
| `$to-tickets` -> `$parallel-implement` | The user explicitly requested top-level delivery of a verified non-empty parent graph; recommend the parent and stop |

Triage remains a peer Ready-contract producer. The relationship index, router,
callers, callees, and tracker owners require no topology edit.
