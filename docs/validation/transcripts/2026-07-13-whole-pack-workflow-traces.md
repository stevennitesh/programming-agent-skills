## Trace 3: Parallel Implementation To Tracker Lock

| Hop | Context loaded | Owner, gate, output, and mutation |
| --- | --- | --- |
| Frontier | Ready items, dependency edges, proof/write scopes, run ledger, engineering contract | `$parallel-implement` orchestrator owns frontier/downshift, slot lock, claims, and run outcome. Only semantically independent ready lanes dispatch. |
| Lane | Complete worker brief, absolute detached worktree, assigned item, proof lane | Direct fresh-context lane worker edits only its checkout and returns one bounded commit or blocker packet. It never mutates tracker state or spawns. |
| Integrate | Accepted worker packet, landing mode/authority, integration checkout, ledger | Integrator lands serially, validates, records exact partial Git state, and returns a review-ready packet. It never dispatches formal review. |
| Conflict | Operation state, three-way sources, resolver authorities | `$resolving-merge-conflicts` owns authorized reconciliation; unresolved or unauthorized state returns blocked without invented landing. |
| Review | Run fixed point, integrated `HEAD`, full integrated diff, authoritative Spec | Orchestrator waits for lane agents to become idle, invokes exactly one review route, and records its result. Ordinary review P0/P1, missing validation, or incomplete Spec keeps Lock closed. |
| Lock | Approved closeout `HEAD`, validation, review, release evidence | Orchestrator owns final outcome and tracker Lock. `partial` or `blocked` cannot claim approved `HEAD`, completed review, push, or no active mutation. |

No unnecessary hop remains: the child integrator returns once; formal review stays root-owned; the resolver is invoked only from actual conflicted/partial Git state.

## Trace 4: Merge-Conflict Resolution

`State -> Trace -> Reconcile -> Prove -> Finish` is the complete workflow.

- Status, explanation, or review loads State and three-way Trace only. Files, index, commits, and operation state stay unchanged.
- A request to resolve supplies reconciliation authority for in-scope conflicts only. Each resolution traces base and both sides, including operation-aware rebase roles.
- Finish authority is separate. Without it, staging and continuation remain untouched.
- Authorized reconciliation completes only when every in-scope entry is reconciled. Any blocked path returns blocked with exact remaining state.
- Abort, hard reset, forced cleanup, or discarding a side remains an explicit destructive boundary.

## Pack-Level Findings From The Traces

- Entry is predictable: setup beats delivery; uncertain intent enters Wayfinder; settled multi-slice source enters ticketing; one ready slice enters serial implementation; independent ready frontier enters parallel orchestration.
- Ownership is singular at mutation boundaries: tracker docs own transport, domain-modeling owns durable language, implementation owners own Lock/commit/closeout, review owns findings only, and conflict Finish remains separate.
- Context repetition is now mostly leading words plus pointers. Ready fields, the diagnosis/TDD fact gate, Spec requirement, mutation read-back, and completion boundaries remain explicit because they are independently entered safety gates.
- Recovery paths are named: setup reconciliation, worker/ledger resume, blocked conflict handoff, installer recovery, review drift rerun, and tracker partial-mutation read-back.
- No routing loop or premature completion path survived the trace. Remaining reduced-confidence areas are live connector partial failure, detached-worktree parallel execution, conflict reconciliation, and full serial Lock execution; their contracts and regression checks pass, but this audit did not mutate those external or Git fixtures.
