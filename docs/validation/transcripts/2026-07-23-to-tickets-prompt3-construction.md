## Intended Contract

To Tickets explicitly transforms one settled source packet into an exhaustive,
dependency-ordered set of bounded implementation tickets satisfying the
configured tracker's Ready-for-agent contract. It owns slicing, commitment
coverage, blocker order, ticket readiness, execution profiles, safe publication
scope, and one stopped downstream recommendation.

Source owners retain product outcome, commitments, acceptance, scope, and
domain meaning. Tracker and label docs retain transport, relationship, role,
lifecycle, and Mutation read-back procedure. The engineering contract retains
shared Source Trace, proof, and state-boundary semantics. `$implement` owns one
selected ticket; `$parallel-implement` owns only an explicitly requested
top-level parent delivery over a complete non-empty graph; `$repo-bootstrap`
owns missing or incompatible setup.

M0 completion requires settled source, complete commitment dispositions,
Ready-for-agent tickets, a complete acyclic ordered graph, per-ticket execution
profiles, applicable state matrices, exact authorized mutation read-back, a
truthful frontier, and one recommendation or `none`. H1 adds only exact
proposal-revision approval, minimum-sufficient information judgment, and
correlated ambiguous-write reconciliation.
