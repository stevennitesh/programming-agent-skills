## Intended Contract

Parallel Implement delivers one explicitly requested, root-owned parent
through its exhaustive associated non-empty Ready-for-agent graph. It may use
qualified concurrent isolated lanes, but owns serial landing, current-HEAD
recombined proof, independent review, bounded Repair, and child-first then
parent-last closeout. The root retains every admission, integration, external
mutation, residual-risk, and completion decision.

Admission rejects delegated invocation before mutation and routes a single
item to `$implement`, an incomplete or unready graph to `$to-tickets`, and
missing setup to `$repo-bootstrap`. A worker or bounded child integrator owns
only its assignment and returns evidence and state to the root. Any
nonterminal return preserves accepted and unrelated state, accounts for
actors, lanes, claims, overlays, and incomplete tracker items, and supplies
the safest recovery or resume action.

Completion requires the drained exhaustive graph, all accepted changes in the
reviewed current integration `HEAD`, final proof, independent review, ordered
tracker closeout with mutation read-back, released claims, safe lane
dispositions, and applicable verified push evidence.
