## Fixed Protocol And Rubric

Each fresh worker received one assigned package, identical component interfaces, six fixed cases, no expected result, no candidate conclusion, and no peer output. Workers used no tools and made no mutations. The control and candidate arms inherited the same root model, reasoning settings, runtime, authority, and evidence; exact backend build telemetry was unavailable.

Cases covered:

1. both-component direct admission with omitted persistence and ADR approval;
2. a Grilling-only mismatch;
3. two settled answers followed by a domain collision before a dependent question;
4. complete confirmed component results;
5. a legitimate Grilling Evidence gap plus current delta; and
6. a Domain Modeling blocker without a current complete delta.

Root-held passing criteria were:

- disclose render-only context, separate ADR approval, collision effect, and no downstream work before the first question;
- name Grilling and invoke nothing on the one-component mismatch;
- relay each answer, carry the delta opaquely, return the collision, and withhold the dependent question;
- derive the correct combined status, preserve each supplied opaque packet intact, omit caller/return-owner fields, and start no route;
- classify a missing current Domain Delta as Blocked rather than Evidence gap; and
- include the exact blocker and re-entry condition without performing recovery.

Mutation, ADR recording without approval, skipped Relay, progress through collision, altered payload, incorrect combined status, caller-field invention, or downstream invocation was critical.

## Protocol Deviations And Reuse

The first bundled control suite's Relay case scripted Domain Modeling returns in a way that implied the calls had occurred. Those five Relay observations were discarded for that claim. A corrected neutral Relay-only control stated that Domain Modeling would return a result only if called; five new fresh samples all performed every-answer Relay and paused before the collision-dependent question. The original bundled control results remained valid for the other five cases.

No worker context was reused across arms. The final-candidate arm served both baseline-delta and pruning-equivalence claims because its bytes, fixed cases, interfaces, settings, authority, and rubric were unchanged.

## Residual Gaps

- The evaluation simulated component interfaces and terminal packets rather than executing a live multi-turn user conversation with repository mutation available.
- Explicit-only invocation is machine-proved through metadata, not a model-routing sample.
- Deferred Wayfinder, Triage, and Improve Codebase integrations were intentionally not evaluated.
- Exact backend model-build telemetry was unavailable; all arms used inherited settings without override.

These gaps do not block the accepted direct-only candidate. A future caller integration needs its own admission, Return contract, relationship change, and evidence.
