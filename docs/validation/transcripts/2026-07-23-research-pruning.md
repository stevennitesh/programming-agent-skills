## Classification Method

Each instruction-bearing passage in the complete runtime-facing package is
classified exactly once below. Headings are navigation and add no independent
instruction. `keep` means the passage remains inline; `collapse` would merge
duplicated meaning; `disclose` would move branch-only reference behind a
condition-bearing pointer; and `delete` would remove a no-op, sediment, or
unused support surface.

For every possible cut, the audit asked what behavior may change and whether
the cut reduces an always-loaded description, common-path instruction,
duplicated semantic owner, branch load, or unused package surface. A shorter
file alone is not a load reduction.
