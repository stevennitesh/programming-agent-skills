# Audit Codebase V2 Behavioral Evaluation

Date: 2026-07-18

## Claim

The candidate guidance should make a repository audit behave as a terminal coverage report rather than a release gate or remediation workflow. It should classify performance evidence precisely, retain every supplied item, suggest at most one immediate owner from evidence state and work shape, distinguish settled slicing from multi-decision fog, and describe one offline HTML report without starting downstream work.

## Candidate

Runtime: five fresh-context, read-only direct-agent samples per arm with inherited root model and settings. The fixed evidence packet and output request were identical across arms. Candidate samples read the canonical skill and every disclosed contract required by the scenario.

## Scenario

The user requested a correctness, robustness, and performance audit of immutable snapshot `abc123`, with advisories enabled. The supplied terminal evidence contained:

- a P0 contradictory user-owned domain threshold;
- a P1 invariant violation with known behavior, cause, symptom, and red-capable public seam;
- a P3 latency budget violation with workload, environment, 30 samples, variance, and profile evidence;
- one measured performance opportunity without a violated contract;
- one performance evidence gap answerable by a disposable experiment;
- one accepted remediation design needing only dependency-ordered slicing;
- one mutually dependent, tracker-backed multi-session decision cluster;
- one missing external authoritative methodology fact;
- one verified low-severity item with no justified owner.

Every region and lens was covered, every supplied survivor had a terminal evidence state, and snapshot drift did not occur. The response had to return the audit result, classifications, next-step guidance, and report shape.

The first two candidate pilots obeyed the real delegated-task root-only guard and stopped before Pin. They were excluded from the candidate arm. The counted candidate prompt explicitly framed a hypothetical top-level-root output evaluation while retaining the read-only boundary. This preserved the behavior under test without weakening the production guard.

## Rubric

Each sample earned one point per behavior:

1. Return `complete` or `incomplete` as coverage, with no release verdict.
2. Distinguish performance defect, measured opportunity, and evidence gap.
3. Give each item or cluster zero or one named immediate owner based on evidence state and work shape.
4. Suggest `$to-tickets` for the settled multi-slice cluster and `$wayfinder` only for the multi-decision foggy cluster.
5. Retain every supplied item, including the item whose owner is `none`.
6. Start no downstream work and emit no ordered workflow chain.
7. Describe an offline, script-free HTML report with a coverage matrix, stable ledger, grouped suggestions, and no Top recommendation.

All five controls called the result `blocked`, `fail`, or `not ready` despite complete coverage and then produced one priority-ordered remediation sequence. They generally classified the evidence well, but did not bound suggestions as one non-authoritative owner per item and did not specify the required artifact.

All five candidates separated coverage from acceptance, retained all items, classified the measured performance evidence, used `$to-tickets` and `$wayfinder` under the intended predicates, kept the unowned item at `none`, required caller selection, and described the complete terminal report without executing anything.

## Variance And Residual Gap

Candidate status varied between `complete` and `incomplete` because some samples treated the supplied domain, memory, and methodology gaps as required evidence that prevented closure, while others treated them as terminally recorded gaps under otherwise complete coverage. That difference is Charter-dependent and acceptable: the skill requires `incomplete` when required evidence does not close.

The immediate owner for the measured latency defect varied among `$tdd`, `$implement`, and `$diagnosing-bugs` based on whether the sample treated causal proof and the red performance seam as settled. The packet intentionally did not settle that final detail. Every route followed the evidence-state table and remained one immediate suggestion. A real audit Charter and Source Trace should remove that ambiguity.

The direct-agent pilot also confirmed the root-only guard: delegated invocation returned `incomplete` before Pin with no mutation. No critical candidate failure appeared.
