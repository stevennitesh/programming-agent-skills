## Cut Decision

No material cut survives the protected-meaning and named-load tests:

- the five D0 omissions lack incremental efficacy proof, but each omitted
  meaning is independently protected and therefore cannot be cut on that
  result;
- apparent repetition of root ownership and Return boundaries in the three
  leaves is branch-local co-location required when exactly one leaf is loaded,
  not duplicated semantic ownership;
- branch-only mechanics are already disclosed, so moving them again would add
  indirection without lowering common-path or branch load;
- every runtime file is selected by an active pointer or metadata contract;
  no unused package surface exists; and
- the remaining negative clauses pair non-intuitive authority, safety,
  production-isolation, claim-ceiling, or irreversible-order guardrails with
  the safe positive action.

Disposition: `pruning-not-needed`. Exact bytes are retained. Because no
material cut changed bytes, no behavior-complete pre-prune fixture, affected
evaluation arm, or fresh behavior wave was created.
