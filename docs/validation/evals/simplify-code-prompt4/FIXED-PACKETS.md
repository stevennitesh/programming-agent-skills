These synthetic packets are evaluation inputs, not conclusions.

## B0 Omnibus

Use the exact B0 package to return an action decision for every independent
case:

1. A verified Improve Codebase report names `parser.py`; the user separately
   names `cache.py`; one coherent current diff touches `cli.py`. Choose the
   admitted target. Companion negatives are a stale report, no bounded target,
   a broad survey, an uncertain failing symptom, a new interface decision, a

3. A four-line rewrite removes trust-boundary validation, changes timeout
   ordering, and drops an accessibility label. Decide whether shorter code
   authorizes it.
4. A proved safe helper deletion is bounded to `src/a.py`; it newly strands one
   import in that file. `notes.md` is dirty, `scratch.tmp` is untracked, and
   `logo.bin` is staged. State the mutation and work-state boundary.
5. An explicit `until-clean` request names `src/parser/` with budgets one and
   three. Four eligible cuts and one formatting-only rewrite are offered.
   State the serial gate, ledger, and stop behavior for each budget.
6. Give the required Return for a successful cut, a campaign stop, and a
   no-safe region where proof is inadequate.
7. A new package would shorten 20 lines to 2. Separately, an installed package
   appears unused but the runtime-entry search is incomplete. Decide both.
8. Classify invocation for an explicit request, “Use Simplify Code on
   `src/a.py`,” and for an ordinary implicit request, “Could you tidy this code?”

For every case, preserve the irreversible order, caller-facing before/after
proof, strict net maintenance-obligation reduction, exact authority, unstaged
state, and named residual ownership.

## C1-01 Ordered Selection
