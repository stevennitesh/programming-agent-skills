# C1-05 Attempt-Owned Recovery — B0 Control Sample 4

## Positive

Recovery may occur by reversing only the exact recorded isolated delta after
refreshing the affected paths and confirming that their current bytes exactly
match the recorded attempt result. The reversal restores the starting bytes,
including the unrelated user edits that were already present.

The unrelated user edits, index, unrelated tracked paths, untracked paths,
repository installation state, trackers, and external state remain untouched.

Return **No safe simplification** and stop. The evidence Return includes the
starting ref and work state, starting-byte identity, recorded isolated delta,
attempt-result byte identity, refreshed current-byte identity, the failing
before/after proof evidence, the recovery reversal, restored-byte identity,
ending work state, changed-path refresh, and confirmation that the index and
unrelated state remain as found.

## Negative: current bytes drift after the attempt

No recovery mutation may occur. Do not reverse the recorded delta against
drifted bytes, because that could overwrite changes not owned by the attempt.

The current bytes, unrelated user edits, index, other tracked paths, untracked
paths, installation state, trackers, and external state remain untouched.

Return **No safe simplification** and stop. The evidence Return includes the
starting-byte identity, recorded isolated delta, attempt-result byte identity,
refreshed current-byte identity, the exact mismatch establishing drift, the
failing proof evidence, unchanged ending work state, and the residual recovery
owner.

## Negative: the attempt cannot be isolated

No recovery mutation may occur. Without an attributable inverse, no bytes may
be rewritten.

All current bytes, unrelated user edits, index, tracked and untracked paths,
installation state, trackers, and external state remain untouched.

Return **No safe simplification** and stop. The evidence Return includes the
starting and ending work state, available before/attempt/current byte
identities, the failing proof evidence, the exact attribution gap that prevents
isolating the attempt, skipped recovery, and the residual recovery owner.

## Negative: an untracked file is ambiguously involved

No recovery mutation may occur. Do not delete, overwrite, move, or otherwise
alter the untracked file, and do not reverse any delta whose correctness
depends on attributing that file to the attempt.

The ambiguous untracked file, unrelated user edits, index, all other tracked
and untracked paths, installation state, trackers, and external state remain
untouched.

Return **No safe simplification** and stop. The evidence Return includes the
starting and ending work state, the untracked path and refreshed identity,
available recorded delta and byte identities, the failing proof evidence, the
exact ambiguity, skipped recovery, and the residual recovery owner.

## Negative: binary state cannot be attributed

No recovery mutation may occur. Do not replace, reconstruct, or remove the
binary state without evidence that identifies the attempt-owned byte delta and
its exact inverse.

The binary bytes, unrelated user edits, index, other tracked and untracked
paths, installation state, trackers, and external state remain untouched.

Return **No safe simplification** and stop. The evidence Return includes the
starting and ending work state, available binary byte identities, the failing
proof evidence, the exact attribution gap, skipped recovery, and the residual
recovery owner.
