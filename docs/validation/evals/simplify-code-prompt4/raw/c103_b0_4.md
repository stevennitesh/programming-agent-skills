# C1-03 Exact Index Identity

## Main condition

**Lock:** Lock does not succeed.

**Before evidence:** Before the cut, record the starting ref, full work status,
relevant diff, staged-state shape, and the exact index identity of
`assets/logo.bin` and `src/config.txt`. Exact identity must include the cached
content identity for each staged entry, such as its index mode, stage, and blob
object ID; paths and status letters alone are insufficient.

**Comparison:** After proof, refresh the work state and compare each current
index entry with its corresponding pre-cut entry. Both cached blob identities
change, so the comparison detects drift even though the paths and `M` status
letters remain unchanged.

**Mutation and state:** The simplification may have produced only its bounded
unstaged cut. Once the index drift is detected, it may not stage, reset,
restore, or otherwise rewrite the concurrent actor's index state, and it may
not make further production, tracker, installed, or external-state changes.
The working-tree state outside the cut remains untouched.

**Return:** Return the bounded region, the attempted cut, preserved contract
and proof seam, before and after proof, changed paths, starting and ending work
state, the exact cached-entry mismatch, the unsuccessful Lock, and drift as the
stop reason. Account for the concurrent index changes separately from the
unstaged cut; do not report a completed simplification.

## Wrong-condition companion

**Lock:** Lock does not succeed.

**Before evidence:** There is no valid pre-mutation snapshot. A snapshot first
taken after mutation is not evidence of the starting index, and status text
does not identify cached bytes.

**Comparison:** Comparing the post-mutation status snapshot with later status
text sees the same paths and `M` letters and therefore cannot detect or exclude
the earlier cached-byte drift. No exact before/after index-identity comparison
is available.

**Mutation and state:** This evidence cannot authorize or lock a production
mutation. Make no code, index, tracker, installed-state, or external-state
change, and do not attempt to reconstruct or restore the unknown earlier index.

**Return:** Return **No safe simplification** with the bounded region, the
missing pre-mutation exact-index snapshot as the proof gap, the observed
post-mutation status-only state, skipped mutation, unchanged invocation-owned
state, and the absent credible index-preservation proof as the exact stop
reason.
