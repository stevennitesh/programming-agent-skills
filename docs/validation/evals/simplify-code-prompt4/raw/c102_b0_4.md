# C1-02 — Simplify Code read-only decision

## Positive

**Decision:** Cut, conditionally licensed.

**Proof boundary:** The authoritative versioned protocol contract fixes the
caller-visible input and output. The smallest caller-facing characterization is
an existing-seam proof: it must pass against the exact starting state and must
fail when the proposed branch deletion changes that observable contract. Once
that red-capable characterization is observed passing before mutation, it can
authorize the bounded branch deletion. The proof does not license changes
outside those caller-visible inputs and outputs.

**State:** Read-only decision only. No cut was made and no baseline or after
proof was run here. Target worktree, index, installed state, and external state
were not changed or claimed; their starting and ending shapes remain to be
recorded by the owning Simplify Code run.

**Return:** Return the licensed candidate to Simplify Code for starting-state
capture, the focused before proof, one unstaged cut, the same after proof,
nearest relevant checks, and net-obligation accounting.

## Negative

**Decision:** **No safe simplification.**

**Proof boundary:** The proposed assertion observes only a private helper's
current branch layout. It cannot detect a change in caller-visible output, so it
is not a behavior-preserving proof seam. Creating a public hook would introduce
a new interface commitment and lies outside this simplification. The exact
proof gap is an existing caller-facing observation that distinguishes preserved
from changed output.

**State:** Read-only decision only. No public hook, test seam, or production cut
was created, and no baseline or after proof was run. Target worktree, index,
installed state, and external state were not changed or claimed.

**Return:** Stop before mutation. Return the new public proof-seam or interface
decision to Codebase Design. If an existing caller-facing seam is later
identified, return a newly bounded candidate to Simplify Code for fresh Trace
and Baseline.
