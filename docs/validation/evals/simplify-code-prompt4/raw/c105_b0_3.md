# C1-05 Attempt-Owned Recovery

## Positive: current bytes exactly match the recorded attempt result

Recovery may occur by applying the exact inverse of the recorded isolated
attempt delta to the current bytes. Do not restore the whole file from the
starting ref: the recovery is limited to attempt-owned hunks. Afterward,
refresh the affected paths and work state and confirm that the recovered bytes
equal the recorded starting bytes.

Unrelated user edits that existed in the starting bytes, the index, unrelated
tracked and untracked paths, pre-existing dead work, installed state, trackers,
and external state remain untouched.

Return **No safe simplification** and identify proof failure as the stop reason.
Include the starting bytes or digest, the recorded isolated delta, the recorded
attempt-result bytes or digest, the pre-recovery equality between current bytes
and that attempt result, the recovered bytes or digest matching the start, the
failed proof output, refreshed changed paths and status, starting and ending
index shape, unrelated-state preservation, zero net obligation reduction, and
the owning residual.

## Negative: current bytes drift after the attempt

Do not apply an inverse delta, restore a file, or otherwise recover through the
drift. The current bytes no longer establish that reversing the recorded
attempt would touch only attempt-owned state.

Leave the drift, unrelated user edits, index, unrelated tracked and untracked
paths, installed state, trackers, and external state untouched.

Return **No safe simplification** with the exact stop reason that refreshed
current bytes differ from the recorded attempt result. Include the starting
bytes or digest, recorded isolated delta, recorded attempt-result bytes or
digest, refreshed current bytes or digest and diff showing the mismatch, failed
proof output, current changed paths and status, starting and ending index shape,
the unresolved recovery risk, and the owner of the drifted work.

## Negative: the attempt cannot be isolated

Do not attempt recovery. Without an exact attempt-owned delta, no inverse edit
is authorized because recovery could remove pre-existing or concurrent work.

Leave all current bytes, unrelated user edits, the index, tracked and untracked
paths, installed state, trackers, and external state untouched.

Return **No safe simplification** with inability to attribute an isolated
attempt delta as the stop reason. Include the starting snapshot or digest,
available before/after diffs and status, the specific attribution ambiguity,
failed proof output, current changed paths, starting and ending index shape,
skipped recovery and its overwrite risk, zero net obligation reduction, and
the owner of the ambiguous work.

## Negative: an untracked file is ambiguously involved

Do not delete, overwrite, move, or incorporate the untracked file, and do not
claim recovery around it. Its provenance is not established as attempt-owned.

Preserve the untracked file byte-for-byte, along with unrelated user edits, the
index, all other tracked and untracked paths, installed state, trackers, and
external state.

Return **No safe simplification** with ambiguous untracked-file ownership as
the stop reason. Include the path, refreshed untracked status, available byte
digest or metadata, the recorded attempt delta and result where available, the
evidence that does not resolve ownership, failed proof output, current changed
paths, starting and ending index shape, skipped recovery and its data-loss
risk, and the residual owner.

## Negative: binary state cannot be attributed

Do not replace, reverse, regenerate, or delete the binary state. Byte changes
whose ownership cannot be attributed cannot be recovered safely.

Preserve the binary bytes, unrelated user edits, the index, all other tracked
and untracked paths, installed state, trackers, and external state.

Return **No safe simplification** with unattributable binary state as the exact
stop reason. Include available starting, attempt-result, and current binary
digests and sizes; the recorded delta or the fact that no trustworthy binary
delta exists; refreshed status; the attribution gap; failed proof output;
current changed paths; starting and ending index shape; skipped recovery and
its corruption or data-loss risk; zero net obligation reduction; and the
residual owner.
