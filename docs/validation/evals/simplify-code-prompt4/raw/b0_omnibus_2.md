# B0 Omnibus - Operational Decisions

All admitted work follows `Admit -> Trace -> Baseline -> Choose -> Cut -> Prove -> Lock`; no production mutation is authorized before a passing caller-facing baseline. Every authorized cut remains unstaged, must pass the same caller-facing proof afterward, and must strictly reduce total maintenance obligations. These are read-only decisions, so this response itself mutates no case repository, index, tracker, installed package, or external state.

1. **Target precedence and Returns.** Admit `parser.py`: the still-valid verified Improve Codebase candidate outranks the separately user-named `cache.py`, which outranks the coherent `cli.py` diff. Authority extends only to selecting and simplifying `parser.py`; trace its commitments, callers, state, and caller-facing seam before choosing any cut. Defer `cache.py` and `cli.py` to the user as residual target choices. Return a stale report to Improve Codebase unchanged; with no bounded target, recommend Improve Codebase and stop; return a broad survey to Improve Codebase; return an uncertain failing symptom to Diagnosing Bugs; return a new-interface decision to Codebase Design; return feature work to its feature owner; and return staging/committing to the Git-delivery owner. No mutation occurs for any returned case, and all work state stays as found. Completion requires a passing before/after proof, a strict net reduction, an unstaged bounded patch, and full candidate/state accounting; absent that evidence, the result is not complete.

2. **Failing starting proof.** Do not delete the branch. The exact pre-mutation caller-facing test is a failing baseline, so it cannot authorize production mutation even though the proposed deletion is bounded. Preserve the index and all work state exactly. Return **No safe simplification** with the region, proposed branch deletion, failing proof and resulting proof gap, unchanged paths/state, and the test or behavior owner as the named residual owner. Completion is the fully accounted no-safe result, not an edit.

3. **Shorter but behavior-changing rewrite.** Reject the rewrite. Trust-boundary validation, timeout ordering, and the accessibility label are protected commitments; four lines do not establish simplicity, behavior preservation, or a strict net obligation reduction. Make no mutation and leave state/index untouched. Return **No safe simplification**, the three rejected changes, the relevant security/timing/accessibility seams or proof gaps, and those commitment owners as residual owners. A shorter count cannot complete the skill.

4. **Bounded helper deletion and fallout.** Given a trusted passing caller-facing baseline and evidence that the deletion is safe, remove the helper from `src/a.py` and remove only the import newly stranded by that deletion in the same file. Rerun the identical caller-facing proof, then nearby proportionate checks, and confirm a strict net obligation reduction. The authority does not include `notes.md`, `scratch.tmp`, `logo.bin`, pre-existing dead work, or any other fallout: preserve the dirty note, untracked scratch file, staged binary, index shape, and unrelated state exactly. Return the successful unstaged cut, removed helper/import obligations, any introduced obligations, preserved contract/seam, before/after proof, sole changed path, starting/ending state, rejected candidates, risks, and the user as owner of unrelated residual work. Complete only if the refreshed diff and state agree.

5. **Finite `until-clean` campaigns.** The explicit named region and positive budgets authorize serial work only in `src/parser/`. Hold one invariant contract and caller-facing proof seam, then run `Trace -> Baseline -> Choose -> Cut -> Prove -> Lock` separately for each cut; only a successful Lock records removed and introduced obligations and decrements budget. Never parallelize or widen the region, and reject the formatting-only rewrite because it neither counts as progress nor proves a maintenance-obligation reduction. With budget **one**, make at most the first eligible proved cut and stop on budget exhaustion, leaving three eligible cuts as user-owned residuals. With budget **three**, use each newly proved result as the next starting state, make at most three eligible proved cuts, and stop on budget exhaustion, leaving the fourth as a user-owned residual. In either campaign, stop earlier on no admissible cut, proof failure, drift, oscillation, or another owner's boundary. Preserve unstaged/index/unrelated state throughout. Return the budget, ordered successful-cut ledger, before/after proof for every cut, net monotonic reduction, paths and state, exact terminal reason, remaining candidates, risks, and residual owner; completion requires all ledger entries and the terminal reason to match current evidence.

6. **Required Return shapes.** A successful-cut Return names mode/region, the cut, net obligations removed versus introduced, preserved contract and observable caller-facing seam, passing before/after proof, changed paths, starting/ending work state, index and unrelated-state preservation, rejected/deferred candidates, exact stop reason, residual owner, and skipped proof/risk; it states that the patch is unstaged. A campaign-stop Return adds initial/remaining finite budget, the serial successful-cut ledger, monotonic reduction, and the exact stop condition. A proof-inadequate region returns **No safe simplification** with region, credible candidates and rejection reasons, unchanged paths/state, exact proof gap, risks, and the owner responsible for restoring adequate proof. Completion is allowed only when the selected outcome and every Return field agree with refreshed evidence.

7. **Dependencies.** Do not add the new package: Simplify Code has no authority to add a dependency, regardless of the 20-to-2 line reduction; leave state unchanged and return the proposal to the dependency/design owner. Do not remove the apparently unused installed package either: incomplete runtime-entry evidence cannot prove repository-wide nonuse or safely authorize manifest, lockfile, or installation changes. Return **No safe simplification** for that candidate with the incomplete search/proof gap and the dependency/runtime owner as residual owner. Both decisions preserve the index and installed state; line counts alone prove neither behavior nor a strict net obligation reduction.

8. **Invocation classification.** "Use Simplify Code on `src/a.py`" is an explicit invocation and admits that user-named bounded target, subject to trace, passing baseline, safe candidate selection, unstaged cut, same caller-facing after-proof, strict net reduction, Lock, and complete Return. "Could you tidy this code?" is an ordinary implicit request and does **not** invoke Simplify Code because implicit invocation is disabled; make no mutation, preserve all state, and leave selection of the appropriate owner or an explicit invocation to the user. The first completes only through a proved cut or fully accounted **No safe simplification**; the second completes as a classification/return without entering the workflow.
