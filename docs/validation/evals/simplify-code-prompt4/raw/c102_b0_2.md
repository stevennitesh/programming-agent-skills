# C1-02 B0 Control Sample 2

## Positive

**Decision: Cut.**

The authoritative versioned protocol contract fixes the preserved
caller-visible input and output. The smallest caller-facing characterization
may serve as the baseline when it passes before mutation and is capable of
detecting the proposed branch deletion. Delete only that branch and its
cut-created fallout.

**Proof boundary:** the existing caller entry, the contract-specified inputs
and outputs, and the proposed branch are inside the boundary. Private branch
layout and any new public hook are outside it. Record the starting ref, status,
relevant diff, and index shape; run the characterization against the exact
starting state; rerun the same characterization after the cut; then run the
nearest relevant tests and proportionate repository checks. Lock only if the
caller-visible results remain contract-identical and the maintenance-obligation
ledger shows a strict net reduction.

**State:** this is a read-only decision. No production mutation was performed,
no index entry was changed, and unrelated, installed, and external state remain
untouched. The concrete starting and ending repository state must be captured
when the cut is executed.

**Return:** one bounded, unstaged branch deletion is licensed subject to the
stated baseline, after-proof, work-state, and strict-net-reduction checks.
Return **No safe simplification** instead if the characterization cannot pass
against the exact starting state, cannot detect the deletion, or the contract
does not fully determine the observed caller behavior.

## Negative

**Decision: No safe simplification.**

The proposed assertion preserves a private implementation shape rather than
caller-visible behavior. Because it cannot distinguish changed caller output,
it is not a credible behavior proof. Creating a public hook would introduce a
new interface and proof-seam commitment, which Simplify Code does not own.

**Proof boundary:** the existing caller-visible behavior and existing public
surface are the maximum admissible boundary. The private helper's branch
layout is not an observable seam, and a new public hook is outside the admitted
region. No baseline described by this case can authorize the branch deletion.

**State:** no cut is authorized. Production paths, the index, unrelated work,
installed state, and external state remain unchanged; only this read-only
decision artifact is created.

**Return:** return **No safe simplification** with the rejected branch deletion,
the semantically inadequate assertion, the missing caller-visible proof, and
the unchanged work state. Return any proposed new public hook or proof-seam
design to `$codebase-design`. A later Simplify Code attempt requires an
existing caller-facing seam and a trusted red-capable baseline that observes
the authoritative caller contract without adding a public commitment.
