# C1-01 Ordered Selection — B0 control sample 1

Mode: one bounded, read-only selection decision.

The order is Delete, Reuse, Standardize, Collapse, then Shrink. The companion
conditions are cumulative so that each case exposes the next rung.

## Positive case

- Selected cut: Delete the expired branch proven unreachable.
- Stop: Delete is the first admissible rung; considering later candidates would
  add no justified selection work.
- Proof boundary: repository-wide entry and caller evidence establishes
  unreachability, and the caller-facing fixture is the before/after observable
  seam.
- State boundary: decision only; no cut, proof command, staging, installation,
  tracker, or external-state mutation is performed.

## Companion 1 — Delete excluded

- Selected cut: Reuse the exact project-owned semantic owner.
- Stop: an authoritative semantic condition excludes Delete; Reuse is the first
  remaining admissible rung.
- Proof boundary: the caller-facing fixture must preserve the same observable
  result through the project-owned owner.
- State boundary: decision only; repository, index, installed state, and
  external state remain outside mutation scope.

## Companion 2 — Reuse also excluded

- Selected cut: Use the runtime-native exact match.
- Stop: Delete is already excluded, and an authoritative compatibility
  condition excludes Reuse; Standardize is the first remaining admissible
  rung.
- Proof boundary: the caller-facing fixture must demonstrate exact behavioral
  equivalence at the supported runtime boundary.
- State boundary: decision only; repository, index, installed state, and
  external state remain outside mutation scope.

## Companion 3 — Standardize also excluded

- Selected cut: Collapse the private abstraction that earns no boundary.
- Stop: Delete and Reuse remain excluded, and an authoritative edge-behavior
  condition excludes Standardize; Collapse is the first remaining admissible
  rung.
- Proof boundary: the caller-facing fixture must cover the preserved edge
  behavior while the private indirection disappears.
- State boundary: decision only; repository, index, installed state, and
  external state remain outside mutation scope.

## Companion 4 — Collapse also excluded

- Selected cut: Shrink the loop while preserving behavior.
- Stop: the four earlier rungs are excluded, including Collapse by the
  authoritative coherence condition; Shrink is the first remaining admissible
  rung.
- Proof boundary: the caller-facing fixture must preserve output, ordering,
  cardinality, and relevant edge behavior.
- State boundary: decision only; repository, index, installed state, and
  external state remain outside mutation scope.

## Companion 5 — Shrink excluded

- Selected cut: **No safe simplification**.
- Stop: the earlier rungs remain excluded and the proof condition leaves Shrink
  without a credible before/after seam; no candidate is admissible.
- Proof boundary: the missing or inadequate caller-facing fixture is the exact
  proof gap. Production mutation is not authorized.
- State boundary: no cut or proof command is run; repository, index, installed
  state, and external state remain outside mutation scope.
