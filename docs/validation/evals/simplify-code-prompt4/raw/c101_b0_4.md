# C1-01 Ordered Selection — B0 Control Sample 4

Mode: read-only selection

## Positive case

Selected cut: **Delete** the expired branch proven unreachable.

Stop: Delete is the first admissible rung. Later Reuse, Standardize, Collapse,
and Shrink candidates are not considered once the ordered hunt admits this
coherent cut.

Proof: The fixture supplies authoritative unreachability for the branch and
states that the candidate is admissible. A real cut would require a trusted
caller-facing proof to pass before and after deletion, followed by the nearest
relevant checks.

State boundary: Decision only. No production code, tests, index entries,
dependencies, commits, trackers, or external systems change.

## Companion case 1

Selected cut: **Reuse** the exact project-owned semantic owner.

Stop: Delete is rejected because the authoritative semantics do not establish
that the branch is expired and unreachable. Reuse is the next rung and is the
first admissible cut, so the hunt stops there.

Proof: The authoritative contract blocks deletion; the existing owner has the
same caller-visible semantics. A real cut would compare the caller-facing
fixture before and after replacement and run the nearest relevant checks.

State boundary: Decision only. No production code, tests, index entries,
dependencies, commits, trackers, or external systems change.

## Companion case 2

Selected cut: **Standardize** on the runtime-native exact match.

Stop: No Delete candidate has authoritative reachability proof. Reuse is
rejected because the project-owned owner does not satisfy the required
compatibility boundary. The runtime-native match is the first admissible rung,
so later rungs are not considered.

Proof: Compatibility evidence excludes Reuse; the runtime-native capability
matches the required contract and supported runtime. A real cut would run the
caller-facing fixture before and after substitution and the nearest relevant
checks.

State boundary: Decision only. No production code, tests, index entries,
dependencies, commits, trackers, or external systems change.

## Companion case 3

Selected cut: **Collapse** the private abstraction that earns no boundary.

Stop: Delete lacks unreachability evidence and Reuse lacks an exact
project-owned semantic match. Standardize is rejected because the native
candidate does not preserve required edge behavior. Collapse is the next and
first admissible rung.

Proof: The edge-behavior contract distinguishes and rejects the native
substitute; the private abstraction contributes no caller-visible boundary. A
real cut would characterize the edge through the caller-facing fixture, observe
it before the cut, and rerun it plus nearby checks after the collapse.

State boundary: Decision only. No production code, tests, index entries,
dependencies, commits, trackers, or external systems change.

## Companion case 4

Selected cut: **Shrink** the loop while preserving behavior.

Stop: Delete, Reuse, and Standardize have no admissible exact match at their
respective proof boundaries. Collapse is rejected because removing the private
abstraction would break a coherence boundary. Shrink is the next and first
admissible rung.

Proof: The coherence boundary requires the abstraction to remain; the
caller-facing fixture covers the loop's outputs and edge cases. A real cut
would run that proof before and after the loop reduction and then run nearby
checks.

State boundary: Decision only. No production code, tests, index entries,
dependencies, commits, trackers, or external systems change.

## Companion case 5

Selected cut: **No safe simplification**.

Stop: The candidates cannot establish a trusted caller-facing proof seam.
Without proof that distinguishes preserved behavior, no rung passes Choose, so
no cut is admitted.

Proof: The exact gap is the absence of an authoritative independent oracle for
the caller-facing behavior. No before/after production proof can therefore be
formed from the fixture.

State boundary: No simplification is performed. Production code, tests, index
entries, dependencies, commits, trackers, and external systems remain
unchanged.
