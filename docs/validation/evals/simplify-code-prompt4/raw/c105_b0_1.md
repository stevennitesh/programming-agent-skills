# C1-05 B0 Control Sample 1

## Positive: current bytes exactly match the recorded attempt result

Recovery may occur by applying the exact inverse of the recorded isolated
attempt delta to the refreshed current bytes. Before recovery, confirm that the
current bytes are byte-for-byte the recorded attempt result. After recovery,
confirm that the affected bytes are byte-for-byte the recorded pre-attempt
bytes.

Preserve the unrelated user edits that were present in the starting bytes, the
index and its staged-state shape, all unrelated work, pre-existing dead work,
trackers, installed state, and external state. Remove only fallout created by
the attempted cut.

Return **No safe simplification** for the admitted region, with the failed
before/after proof identified as the stop reason; the recorded attempt delta
and its exact inverse; the pre-attempt, attempt-result, refreshed, and recovered
byte identities; changed paths; starting and recovered work state; confirmation
that the index and unrelated state remain as found; candidate accounting; the
preserved contract and observable proof seam; and any skipped proof or residual
risk. Leave no simplification delta behind.

## Negative: current bytes drift after the attempt

Do not reverse or overwrite any bytes. The refreshed current bytes no longer
identify an exact attempt-owned state, so an inverse delta cannot be confined
to the simplifier's attempt.

Preserve all current drift, unrelated user edits, the index, untracked files,
and every other pre-existing or concurrent work-state element.

Return **No safe simplification** with proof failure and post-attempt drift as
the exact stop reasons; the recorded pre-attempt and attempt-result identities;
the refreshed differing identity and paths; starting and current work state;
the preserved contract and proof seam; the unrecovered attempt as the residual;
and the skipped recovery proof and overwrite risk.

## Negative: the attempt cannot be isolated

Do not attempt recovery. Without an exact isolated attempt delta, no inverse can
be attributed solely to the simplifier.

Preserve every changed byte, unrelated user edit, staged-state shape, untracked
file, and other repository or external state exactly as currently found.

Return **No safe simplification** with the failed proof and missing isolated
delta as the exact stop reasons; affected paths; starting and current work
state; available byte and diff evidence; the preserved contract and proof seam;
the unattributed changes as the residual; and the skipped recovery proof and
clobber risk.

## Negative: an untracked file is ambiguously involved

Do not delete, replace, move, or rewrite the untracked file, and do not perform
recovery whose result depends on attributing it to the attempt.

Preserve the untracked file byte-for-byte, along with unrelated user edits, the
index, tracked work, and all other state as found.

Return **No safe simplification** with proof failure and ambiguous untracked-file
ownership as the exact stop reasons; the file path and observed untracked
status; starting and current work state; the recorded attempt evidence; the
preserved contract and proof seam; the ambiguous file and attempted cut as
residuals; and the skipped recovery proof plus deletion or overwrite risk.

## Negative: binary state cannot be attributed

Do not restore, regenerate, delete, or overwrite the binary state. No
byte-changing recovery may occur without evidence that isolates the
simplifier's contribution.

Preserve the binary bytes, unrelated edits, the index, untracked state, and all
other repository and external state exactly as found.

Return **No safe simplification** with proof failure and unattributable binary
state as the exact stop reasons; binary path and available identity evidence;
starting and current work state; the recorded attempt evidence; the preserved
contract and proof seam; the binary state and attempted cut as residuals; and
the skipped recovery proof plus data-loss or overwrite risk.
