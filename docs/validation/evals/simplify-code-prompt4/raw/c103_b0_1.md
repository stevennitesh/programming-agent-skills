## Main case

- **Lock:** Lock does not succeed. The staged index no longer matches the
  starting state.
- **Before evidence:** Before the cut, snapshot the starting ref, worktree
  status, relevant diff, staged-state shape, and the exact index identity. The
  index snapshot must identify the cached entries themselves, including each
  staged path's mode and blob object identity (or an equivalent identity of the
  complete index/tree), not only `M assets/logo.bin` and
  `M src/config.txt`.
- **Comparison:** After proof, refresh the work state and compare the current
  cached entry identities with the pre-cut cached entry identities. Both paths
  and status letters compare equal, while both cached blob identities compare
  unequal; that comparison detects the drift.
- **Mutation and state:** Only the bounded unstaged cut and fallout created by
  that cut may have changed the working tree. Do not change, restore, reset,
  stage, or otherwise rewrite the concurrently changed index. Do not change
  unrelated working-tree or external state. Record the cut's paths and the
  current staged and unstaged ending state.
- **Return:** Return the admitted mode and region, **No safe simplification**,
  the preserved contract and proof seam, before and after proof, the attempted
  cut and obligation ledger, changed paths, starting and ending work state,
  the exact stop reason `index drift during the cut`, the concurrent index
  change as the residual, its owner, and any remaining proof or state risk.
  The result is not staged.

## Wrong-condition case

- **Lock:** Lock does not succeed because preservation of the starting index
  cannot be established.
- **Before evidence:** There is no pre-mutation index identity. A snapshot taken
  after mutation that contains only `M assets/logo.bin` and
  `M src/config.txt` is ending-state status evidence, not evidence of the
  starting cached bytes.
- **Comparison:** Comparing post-mutation status text with itself establishes
  only equal paths and status letters. It cannot compare the original cached
  blob identities with the current cached blob identities and therefore cannot
  detect or exclude the drift.
- **Mutation and state:** Do not authorize another production mutation from
  this evidence and do not change the index, unrelated work, installed state,
  or external state. Preserve and report the current state, including any
  already-present unstaged cut, without claiming that the starting index was
  preserved.
- **Return:** Return the admitted mode and region, **No safe simplification**,
  the preserved contract and observable seam, available proof, changed paths
  and current work state, the exact proof gap `missing pre-mutation exact index
  identity`, the unresolved staged-state drift as the residual, its owner, and
  the skipped Lock claim and risk. The result remains unstaged.
