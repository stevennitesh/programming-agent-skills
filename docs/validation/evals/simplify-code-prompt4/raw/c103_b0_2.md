# C1-03 B0 Control Sample 2

## Main condition

**Lock:** Lock does not succeed. The staged paths and status letters have the
same shape, but the cached contents no longer have the identity recorded at
the starting state.

**Before evidence:** Before mutation, snapshot the exact index identity, not
only porcelain status: the staged entries for `assets/logo.bin` and
`src/config.txt`, including each entry's blob object ID, mode, and stage, or an
equivalent complete index/tree identity. Also retain the starting status and
staged-state shape so the identity evidence is tied to the recorded starting
work state.

**Comparison:** Refresh the index evidence after proof and compare the cached
entry identities with the before snapshot. The changed blob object IDs, or a
changed complete tree/index identity, detect the drift even though both
snapshots would still describe `M assets/logo.bin` and `M src/config.txt`.
Status-path equality is not an identity comparison.

**Mutation and state:** Simplify Code may change only the admitted unstaged cut
and fallout created by that cut. It may not reset, restore, restage, or
otherwise rewrite either cached entry, and it may not change unrelated
working-tree, tracker, installed, or external state. The concurrent actor's
index changes are left intact and reported; they cannot be absorbed into the
cut.

**Return:** Return the bounded region and cut, the preserved contract and proof
seam, the before and after proof, the exact index-identity mismatch, the
starting and ending work state, changed paths attributable to the cut, and the
index drift as the stop reason. State that Lock did not complete and identify
the staged-state owner as the residual owner.

## Companion wrong-condition

**Lock:** Lock does not succeed.

**Before evidence:** A snapshot first taken after mutation is not before
evidence, and status text records neither cached bytes nor their prior
identity. No comparison can reconstruct the starting cached identities from
that snapshot.

**Comparison:** Comparing post-mutation status text with later status text can
show only that the same paths and letters remain. It cannot detect or exclude
the already-completed cached-byte changes.

**Mutation and state:** No further production mutation is authorized by this
baseline. Simplify Code must not alter the index or unrelated state; any
already-created bounded unstaged cut and the concurrent index state are
accounted for as found rather than repaired or rewritten.

**Return:** Return **No safe simplification** with the missing pre-mutation
exact-index snapshot as the proof gap, the post-mutation status-only snapshot,
the current work state, any already-created cut paths, the inability to
establish index preservation as the stop reason, and the staged-state owner as
the residual owner.
