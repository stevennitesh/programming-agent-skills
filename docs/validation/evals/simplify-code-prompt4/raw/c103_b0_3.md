# C1-03 B0 Control Sample 3

## Main case

- **Lock:** Lock does not succeed. The concurrent cached-byte changes are index drift even though the staged paths and status letters remain unchanged.
- **Before evidence:** Before the cut, snapshot the exact index identity for both staged entries: path, stage, mode, and cached blob object ID (or an equivalent complete index/tree identity derived from those entries). Also retain the starting ref, status, and relevant diff required by the work-state record.
- **Comparison:** After proof, compare the current index entries or exact written-tree identity with the pre-mutation snapshot. The changed cached blob object IDs for `assets/logo.bin` and `src/config.txt` detect the drift; comparing path/status text does not.
- **Mutation and state:** Simplify Code may make only its bounded unstaged cut and cut-created fallout. It must not alter, restore, stage, or otherwise reconcile the concurrently changed index, and it must not change unrelated working-tree or external state. Once drift is found, it performs no further production mutation. Any already-made unstaged cut remains unstaged and is reported separately from the concurrent index changes.
- **Return:** Return the bounded region and mode, the attempted cut, preserved contract and proof seam, before/after proof, changed paths, starting and ending work state, the pre-mutation index identity, the detected cached-object drift, the failed Lock/stop reason, residual owner, and any remaining risk. Do not claim completion.

## Companion wrong-condition

- **Lock:** Lock does not succeed. A snapshot first taken after mutation cannot establish that the index was preserved from the starting state.
- **Before evidence:** There is no valid before-mutation index snapshot. The available status-only text records paths and status letters after the drift and contains neither the prior nor current cached-byte identity.
- **Comparison:** Repeating the status-text comparison yields the same `M assets/logo.bin` and `M src/config.txt`, so it cannot detect or exclude the cached-byte changes. No later comparison can reconstruct the missing starting index identity.
- **Mutation and state:** The inadequate baseline cannot authorize a production simplification. Simplify Code changes no repository, index, tracker, installed, or external state. It also does not attempt to restore the concurrent actor's index changes.
- **Return:** Return **No safe simplification** with the bounded region, the exact proof gap—late, status-only snapshots cannot prove starting-index preservation—the observed current work state, skipped mutation, exact stop reason, and the owner of the unresolved index state.
