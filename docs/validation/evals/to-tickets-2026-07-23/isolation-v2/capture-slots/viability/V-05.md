- Readiness: `ready-for-agent`
- Role: `enhancement`
- Assignee: unassigned
- Work-unit form: independently completable vertical behavior slice.
- Bounded slice: implement the analysis-session restore behavior that selects and completes the correct safe lifecycle action for an absent, current, incompatible, corrupt, or expired persisted snapshot.
- Source Trace: parent `P-505`; source `SRC-V05@1`; commitments `C1`–`C4`; source acceptance “each branch has observable proof” and “corrupt data never becomes current”; proof seam `state fixture snapshot_restore_matrix`.
- Desired behavior:
  - With no snapshot, initialize a valid current-schema session snapshot.
  - With a valid current-schema snapshot, reuse it as the current analysis session.
  - With an incompatible snapshot, reject it before it can become current and rebuild a valid current-schema snapshot.
  - With an expired snapshot encountered after restart, invalidate it before rebuilding a valid current snapshot.
- Edge and error behavior:
  - Corrupt or undecodable persisted data is rejected and never installed as current.
  - A snapshot that is both stale and otherwise decodable still follows the expired branch and is invalidated before rebuilding.
  - Failure to validate an existing snapshot must not fall through to the reusable-current branch.
  - Cross-account lookup, reuse, and sharing are outside this slice and must not be introduced.
- Observable acceptance:
  1. `snapshot_restore_matrix` proves absent state initializes a current-schema snapshot.
  2. It proves a valid current-schema snapshot is reused without rebuilding it.
  3. It proves incompatible and corrupt snapshots are rejected before current-session installation and are replaced with valid current-schema state.
  4. It proves an expired snapshot discovered after restart is invalidated before replacement.
  5. Every supported branch exposes the selected lifecycle action and resulting current-session state to the fixture.
  6. No corrupt input becomes current.
- Relevant seams: analysis-session restore entrypoint; snapshot read/validation seam; schema-compatibility check; expiry check after restart; invalidation/rebuild seam; current-session installation seam; `snapshot_restore_matrix`.
- Expected durable write scope: only the persisted snapshot and current-session state for the same analysis session. The absent branch may create current-schema state; incompatible, corrupt, and expired branches may invalidate/replace that state; the reusable-current branch performs no replacement write. No cross-account data or unrelated session state may be read or written.
- Scope fence: analysis-session snapshot lifecycle only. Excludes cross-account sharing, changes to unrelated session behavior, new persistence formats not required by `SRC-V05@1`, and implementation beyond the restore path.
