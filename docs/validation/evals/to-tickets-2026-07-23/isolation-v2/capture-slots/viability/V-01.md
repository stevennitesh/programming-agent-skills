- Dependency state: true blockers `none`.
- Stable tracker order: 1.
- Proof lane: repository integration lane using `checkout_saved_address`, exercising valid persistence, invalid-postal rejection without persistence, and buyer ownership.
- Verification authority: the repository integration fixture `checkout_saved_address` is the settled source authority.
- Verification evidence: a passing fixture run showing a valid current-buyer address is persisted and reusable, plus a passing invalid-postal branch showing no persistence. Evidence must also show no cross-buyer exposure in the exercised buyer boundary.
