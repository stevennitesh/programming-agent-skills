## Resumable state

**Outcome:** `blocked`, nonterminal, resumable. The immediate primary blocker is
the preserved C integration conflict. Independent retained blockers are A's
foreign-owned claim and B's escaped Python provenance; D is retained
accepted-unlanded behind C and requires stale-base revalidation after C lands.
No overlay currently exists. [A.observations[3]; A.observations[5];
A.observations[9..10]; Runtime:`SKILL.md` nonterminal rule]

A valid runtime-contract-3 checkpoint may be simulated only after all actors
are idle; the current repository-backed HEAD is nonempty; the integration state
is clean or explicitly preserved; every lane is `removed`,
`provider-preserved`, or an explicitly accepted residual; the continuation,
next frontier, exact blockers, tracker and remote observations are recorded;
and every claim is uniquely accounted as retained (owner, token, claim time,
recovery owner) or released with read-back. Those checkpoint facts are not
supplied, so this capture must not fabricate a checkpoint receipt. On a later
resume, authority remains closed until Git, worktrees, actors, tracker, and
remote observations are reconciled. [Runtime:`RUN-LEDGER.md` Checkpoint;
Runtime:`run_ledger.py` checkpoint and resume/reconcile validation]

The safest resume order is: reconcile and route the preserved C conflict;
establish or remove C's overlay from proof; stale-base-check and then
serially land D if authorized; recover B provenance and dispatch it; reacquire
A only after exact ownership changes; then rederive the frontier and continue
serial integration. A and B remain mutually nonconcurrent throughout.
