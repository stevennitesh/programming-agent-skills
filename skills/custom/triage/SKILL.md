# Triage

Run the **intake state machine** for raw issues and configured external PR requests.

Tracker docs own transport, the Ready-for-agent contract, and label mechanics. Triage owns classification, request verification, shaping, state mutation, ready briefs, and rejected-enhancement memory. `$implement` and `$parallel-implement` own code changes and `implemented` closeout.

Apply PR triage only when `docs/agents/issue-tracker.md` enables PRs as a request surface. A named PR bypasses only external-author discovery. For triage, a **PR is an issue with attached code**.

## Contracts

Read `docs/agents/issue-tracker.md` and `docs/agents/triage-labels.md` before tracker access. A missing contract blocks triage; recommend `$setup-matt-pocock-skills`.

When codebase context matters, follow `docs/agents/domain.md`. Before reproduction, checkout, or executable validation, read `docs/agents/engineering-contract.md` when present.

Read [AGENT-BRIEF.md](AGENT-BRIEF.md) only when preparing a `ready-for-agent` or `ready-for-human` brief. Read [OUT-OF-SCOPE.md](OUT-OF-SCOPE.md) only when screening, rejecting, or reconsidering an enhancement.

Treat `$to-tickets` output as already ready-for-agent. Re-enter triage only when the maintainer explicitly asks to correct its state or brief.

**Disclaimer gate:** prefix every issue or comment posted during triage with:

```markdown
> *This was generated by AI during triage.*
```

## State Machine

**Role invariant:** every completed item carries exactly one mapped category role and one mapped state role. Surface missing or conflicting roles before mutation.

Normal intake moves from unclassified to `needs-triage`, then to `needs-info`, `ready-for-agent`, `ready-for-human`, or `wontfix`. Reporter activity returns `needs-info` to `needs-triage`.

For a PR, `ready-for-agent` means a Codex-ready brief names the next bounded slice on the diff. `ready-for-human` means the PR is ready for human judgment or merge.

`implemented` is owned by `$implement` or `$parallel-implement`. Triage may apply it only as an explicit label correction backed by existing closeout evidence.

**Ready gate:** `ready-for-agent` satisfies the tracker's Ready-for-agent contract and adds verification evidence. Record an explicit label-only override in the triage packet.

## Choose One Branch

- **Attention scan:** show incoming work that needs maintainer attention. Read [ATTENTION-SCAN.md](ATTENTION-SCAN.md) completely.
- **Specific item:** verify, shape, recommend, and apply one outcome. Read [SPECIFIC-ITEM.md](SPECIFIC-ITEM.md) completely.
- **Quick override:** apply an explicitly named state outcome with reduced discovery. Read [QUICK-OVERRIDE.md](QUICK-OVERRIDE.md) completely.

## Completion Criteria

Attention scan completes only through its read-only branch criterion.

A mutation branch completes only when the maintainer-approved packet was applied, read back, and returned as a triage packet; role invariants hold; every tracker comment carries the disclaimer; required briefs and rejected-enhancement memory exist; and partial failures, skipped checks, and blockers are reported.
